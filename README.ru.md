# VaultClip

Менеджер буфера обмена для macOS с шифрованием истории на диске, вкладками «Избранное» и «Пароли» и фильтрацией копий из менеджеров паролей.

Форк open source проекта [Yippy](https://github.com/mattDavo/Yippy) (Matthew Davidson). Оригинальная идея и UX сохранены; в этом форке добавлены усиленная безопасность, актуальные тулчейны и переименование в **VaultClip**.

**English:** [README.md](README.md)

## Возможности

### История буфера обмена

- **Фоновый мониторинг** системного pasteboard: каждая новая копия (кроме отфильтрованных источников) сохраняется в локальную историю.
- **До 5000 элементов** в памяти модели; в настройках можно ограничить отображаемый размер (50–1500).
- **Поддерживаемые типы данных:**
  - обычный текст, RTF и HTML;
  - URL и ссылки;
  - цвета (swatch в списке);
  - растровые изображения (TIFF/PNG и др.);
  - PDF (миниатюра и подпись);
  - файлы с иконкой Finder или превью-миниатюрой.
- **Дедупликация** — повторная копия того же содержимого в пределах последних 20 элементов не создаёт дубликат.
- **Иконка приложения-источника** у строки истории (bundle id копирующего приложения).
- **Метка времени** копирования для элементов без текстового представления.

### Вкладки History / Favorites / Passwords

Панель истории разделена на три вкладки с иконками:

| Вкладка | Назначение |
|--------|------------|
| **History** | Полная хронология (без избранного и сохранённых паролей — они скрыты из общего потока). |
| **Favorites** | Закреплённые элементы; не удаляются при обычной очистке истории и не вытесняются лимитом так же, как обычные записи. |
| **Passwords** | Явно сохранённые пароли с опциональным комментарием (сайт, логин, заметка). |

Избранное и сохранение в «Пароли» доступны **только через контекстное меню** (ПКМ по строке):

- добавить / убрать из избранного;
- сохранить в пароли (с запросом комментария);
- убрать из паролей;
- изменить комментарий к паролю;
- удалить элемент.

### Поиск и превью

- **Поиск по строке** (⌘\\ — фокус в поле поиска): нечёткое сопоставление по тексту элементов.
- Для **сохранённых паролей** в общей истории поиск идёт **только по комментарию**, не по значению пароля.
- **Превью** (Ctrl+Space): отдельное окно с текстом, изображением или Quick Look для файлов; для паролей вне вкладки Passwords содержимое **маскируется**.

### Вставка и навигация

- Выбор элемента и **Return** — вставка в приложение, которое было активно до открытия панели (симуляция ⌘V через Accessibility).
- **⌘0 … ⌘9** — быстрая вставка элемента по позиции в текущем списке.
- **Перетаскивание** строк для изменения порядка; перенос на первое место обновляет системный буфер.
- Настройки **показа и вставки rich text** (RTF/HTML) независимо друг от друга.

### Окно и позиция панели

Приложение живёт в **строке меню** (без иконки в Dock). Панель истории можно закрепить:

- слева / справа / сверху / снизу экрана;
- по центру (несколько размеров) или на весь экран.

Позиция запоминается в настройках. Переключение: меню **Position** или **Ctrl+Alt+⌘ + стрелки** при открытой панели.

### Меню строки состояния

- **Toggle Window** — показать/скрыть панель (по умолчанию **⌘⇧V**, настраивается в Preferences → Hot Key).
- **Launch at Login** — автозапуск при входе в систему.
- **Delete Selected** / **Clear history** — удаление с выбором: только история или всё включая избранное и пароли.
- **Preferences**, **Help**, **About**.

### Первый запуск

При первом открытии показывается **Welcome** с запросом доступа **Accessibility** (нужен для вставки ⌘V в другие приложения). Без этого разрешения история работает, но автоматическая вставка — нет.

## Безопасность и конфиденциальность

VaultClip хранит всё локально на Mac. Ни история, ни ключи не отправляются в сеть.

### Шифрование на диске

- Содержимое элементов истории записывается в **AES-GCM** (CryptoKit, ключ 256 бит).
- Формат файла: префикс `VC1` + sealed box (nonce ‖ ciphertext ‖ tag).
- Симметричный ключ хранится в **связке ключей macOS**:
  - service: `VaultClip`, account: `history-data-key`;
  - `kSecAttrAccessibleAfterFirstUnlockThisDeviceOnly`, без iCloud sync;
  - при создании ключа запрашивается **user presence** (Touch ID / пароль входа).
- **Метаданные** (избранное, флаг пароля, комментарий, время копирования, bundle id источника) тоже пишутся через тот же шифрующий слой.
- Старые **незашифрованные** файлы читаются при открытии; при следующей записи перешифровываются. Если расшифровка не удаётся, читаются сырые байты (защита от потери данных при смене ключа).

### Права на файловую систему

- Каталог истории: `~/Library/Application Support/VaultClip/history/` с правами **0700** (только владелец).
- Перед записью проверяется, что путь **не symlink** и лежит внутри ожидаемого Application Support.
- Имена файлов типов pasteboard **санитизируются** (кодирование `%`, `/`, `:`).

### Фильтрация чувствительных копий

Копии из известных **менеджеров паролей** не попадают в историю (по bundle id приложения-источника), в том числе:

1Password, LastPass, Bitwarden, Dashlane, Keeper, Elpass, Keychain Access, Apple Passwords, Proton Pass, NordPass, Enpass и др.

Дополнительно отбрасываются служебные типы pasteboard (transient, concealed, auto-generated и vendor-specific UTI менеджеров паролей).

### Пароли в интерфейсе

- Элементы, явно сохранённые во вкладку **Passwords**, в **History** и **Favorites** отображаются как `••••••••`, не участвуют в поиске по значению.
- Во вкладке **Passwords** показывается реальное значение и комментарий.
- **Превью** маскирует пароли вне вкладки Passwords.
- При вставке пароля — **предупреждение**; в pasteboard добавляется тип `org.nspasteboard.ConcealedType`, где это уместно.

### Среда выполнения и сеть

- Включён **Hardened Runtime** для всех конфигураций сборки.
- **App Transport Security**: произвольные HTTP-загрузки запрещены (`NSAllowsArbitraryLoads = false`). Приложение не использует сетевые API для истории.
- **App Sandbox** в текущей сборке **не включён** (требует отдельного решения по entitlements для clipboard/accessibility при подписи Developer ID).

### Разрешения macOS

| Разрешение | Зачем |
|------------|--------|
| **Accessibility** | Однократная симуляция ⌘V в ранее активное приложение. Keystroke logging не ведётся. |
| **Keychain** | Хранение ключа шифрования; при отказе показывается диалог с переходом в Keychain Access. |

Описания для системных диалогов — в `VaultClip/Supporting Files/Info.plist`.

### Где лежат данные

```
~/Library/Application Support/VaultClip/
├── history/          # зашифрованные элементы (по UUID)
├── error.log
└── warning.log
```

Настройки — `~/Library/Preferences/VaultClip.plist` (через UserDefaults).

**Миграция с Yippy:** при первом запуске после обновления данные переносятся из `~/Library/Application Support/MatthewDavidson.Yippy/`, настройки и ключ Keychain копируются автоматически.

Удаление записи `history-data-key` для VaultClip в Keychain Access **необратимо ломает** расшифровку старой истории; приложение создаст новый ключ и начнёт писать заново.

## Горячие клавиши

Действуют при **открытой** панели истории (часть — глобально):

| Сочетание | Действие |
|-----------|----------|
| **⌘⇧V** (по умолчанию) | Открыть / закрыть панель |
| **↑ / ↓** | Предыдущий / следующий элемент |
| **Page Up / Page Down** | Прокрутка списка |
| **Return** | Вставить выбранное |
| **Esc** | Закрыть панель |
| **⌘0 … ⌘9** | Вставить элемент по индексу |
| **Ctrl+Delete** | Удалить выбранное |
| **Ctrl+Space** | Превью вкл/выкл |
| **⌘\\** | Фокус в поиск |
| **Ctrl+[** / **Ctrl+]** | Предыдущая / следующая вкладка |
| **Ctrl+Alt+⌘←→↑↓** | Позиция панели на экране |

Горячую клавишу открытия панели можно сменить в **Preferences → Hot Key**.

## Установка

1. Скачайте `VaultClip.dmg` из релизов репозитория **или** соберите из исходников (ниже).
2. Перетащите `VaultClip.app` в «Программы».
3. При первом запуске разрешите **Accessibility** (см. Welcome / System Settings → Privacy & Security → Accessibility).
4. При запросе Keychain — разрешите доступ к ключу шифрования.

### Сборка из исходников

Требования: Xcode, macOS 10.15+, CocoaPods.

```bash
cd /path/to/VaultClip
pod install
open VaultClip.xcworkspace
```

Схемы:

- **VaultClip** — релиз (`VaultClip.app`);
- **VaultClip Beta** — бета (`VaultClip Beta.app`);
- **VaultClip XCTest** — unit- и UI-тесты.

Release-сборка:

```bash
xcodebuild -workspace VaultClip.xcworkspace -scheme VaultClip -configuration Release build
```

### Создание DMG

Установите [create-dmg](https://github.com/andreyvit/create-dmg), положите `VaultClip.app` рядом с `create-installer.sh`:

```bash
./create-installer.sh VaultClip
```

В той же папке появится `VaultClip.dmg`.

## Разработка

Контрибуции приветствуются: issues, pull requests, отзывы.

Структура проекта:

- `VaultClip/Sources/` — код приложения;
- `VaultClipTests/` — unit-тесты (58 тестов);
- `VaultClipUITests/` — UI-тесты (требуют Accessibility в настройках системы).

### Ключевые модули безопасности

- `VaultClip/Sources/Models/Security/` — Keychain, AES-GCM, миграция данных;
- `SecureStorageHelper` — права каталогов и проверка путей;
- `DataFileManager` — префикс `VC1` и шифрование payload;
- `History` — denylist bundle id и типов pasteboard.

- Переименование Yippy → VaultClip (код, bundle id, пути, схемы Xcode).
